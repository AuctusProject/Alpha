Alpha Api

Auctus plataform API for alpha version.
The base API URL is: https://alpha.auctus.org/api/public/v1/

Current version: v0.1

Without Authorization

List Assets

List all available assets.

Request

Content-type: application/json
Complete route: https://alpha.auctus.org/api/public/v1/assets
Verb: GET

Response

Type: JSON
Expected HTTP status codes:

200OKWhat you want.

When status code is 200, the content body will be:

[
	{ 
		id: (integer),
		code: (string),
		name: (string)
	},
	...
]
  • id: Internal identifier for asset.
  • code: Asset code/abreviation.
  • name: Asset name.

With Authorization

Common Behavior

Common behavior for all API calls with authorization.

Request

Content-type: application/json
Base route: https://alpha.auctus.org/api/public/v1/{YOUR_API_KEY}/

YOUR_API_KEY is generated through Alpha plataform. Login and go to https://alpha.auctus.org/manage-api.

Response

Type: JSON
Expected HTTP status codes:

200OKWhat you want.
400Bad RequestSome request data is incorrect.
403ForbiddenInvalid API key.
429Too Many RequestAPI can block if there are many requests for a method using same API key. You should call a specific method only once in each 10 minutes.

List Portfolios

List your own portfolios with all informations, even disabled advisors. Advisors data and assets distribution listed are always the last version.

Request

Complete route: https://alpha.auctus.org/api/public/v1/{YOUR_API_KEY}/portfolios
Verb: GET

Response

When status code is 200, the content body will be:

[
	{
		id: (integer),
		risk: (integer),
		projectionValue: (double),
		pessimisticValue: (nullable double),
		optimisticValue: (nullable double),
		advisor:
		{
			id = (integer),
			name = (string),
			description = (string),
			price = (double),
			period = (integer),
			enabled = (boolean)
		},
		distribution:
		[
			{
				asset: 
				{
					id = (integer),
					name = (string),
					code = (string)
				},
				percent = (double)
			},
			...
		]
	},
	...
]
  • id: Internal identifier for portfolio.
  • risk: Risk defined for the portfolio. (1 - Very Low, 2 - Low, 3 - Medium, 4 - High, 5 - Very High)
  • projectionValue: Daily projection for portfolio (value in percentage multiplied by 100).
  • pessimisticValue: Daily pessimistic projection for portfolio (value in percentage multiplied by 100).
  • optimisticValue: Daily optimistic projection for portfolio (value in percentage multiplied by 100).
  • advisor.id: Internal identifier for advisor.
  • advisor.name: Advisor name.
  • advisor.description: Advisor description.
  • advisor.price: Advisor price in AUC.
  • advisor.period: Period in days to terminate and validate the performance when the advisor was purchased.
  • advisor.enabled: Define if Advisor is available for be purchased by new users. When disabled, only not allow new purchases.
  • distribution.asset: Same fields defined in assets route.
  • distribution.percent: Asset percentage multiplied by 100.

Create Portfolio

Allow creating a new portfolio for an existing advisor. The new portfolio must be for a different risk than risks already defined for existing portfolios from respective Advisor.

Request

Complete route: https://alpha.auctus.org/api/public/v1/{YOUR_API_KEY}/portfolios
Verb: POST

Response

When status code is 200, the content body will be:

{
	advisorId: (integer),
	risk: (integer),
	projectionValue: (double),
	pessimisticValue: (nullable double),
	optimisticValue: (nullable double),
	distribution:
	[
		{
			assetId: (integer),
			percent = (double)
		},
		...
	]
}
  • advisorId: Internal identifier for advisor.
  • risk: Risk defined for the portfolio. (1 - Very Low, 2 - Low, 3 - Medium, 4 - High, 5 - Very High)
  • projectionValue: Daily projection for portfolio (value in percentage multiplied by 100).
  • pessimisticValue: Daily pessimistic projection for portfolio (value in percentage multiplied by 100).
  • optimisticValue: Daily optimistic projection for portfolio (value in percentage multiplied by 100).
  • distribution.assetId: Internal identifier for asset.
  • distribution.percent: Asset percentage multiplied by 100.

New Assets Distribution

Allow creating a new portfolio for an existing advisor. The new portfolio must be for a different risk than risks already defined for existing portfolios from respective Advisor.

Request

Complete route: https://alpha.auctus.org/api/public/v1/{YOUR_API_KEY}/portfolios/{PORTFOLIO_ID}/distribution
Verb: POST

PORTFOLIO_ID is an internal identifier for the portfolio that is intended update.

Content:

[
	{
		assetId: (integer),
		percent = (double)
	},
	...
]
  • assetId: Internal identifier for asset.
  • percent: Asset percentage multiplied by 100.

Note: all assets distribution percentage must match 100% and an asset can be presented only once inside the list.

Response

When status code is 200, the body will be an empty JSON.

Update Advisor

Allow updating pieces of information about Advisor.

Request

Complete route: https://alpha.auctus.org/api/public/v1/{YOUR_API_KEY}/advisors/{ADVISOR_ID}
Verb: PATCH

ADVISOR_ID is an internal identifier for the advisor that is intended update.

Content:

{
	description = (string),
	price = (double),
	period = (integer),
	enabled = (boolean)
}
  • description: Advisor description.
  • price: Advisor price in AUC.
  • period: Period in days to terminate and validate the performance when the advisor was purchased.
  • enabled: Define if Advisor is available for be purchased by new users. When disabled, only not allow new purchases.

Response

When status code is 200, the body will be an empty JSON.
